# Configure DNS settings

import ResetConnectivityWarningPartial from './partials/_reset-connectivity-warning.mdx'

You have the option to configure your own DNS settings, no matter if it's a public or a private server.
DNS settings can be changed in the emnify Portal.
By default, emnify uses the Google DNS 8.8.8.8 and 8.8.4.4.

:::note
Utilizing a private DNS server that isn't accessible via the public internet requires establishing a private network with the device or a network where the private DNS server is situated.
This can be accomplished using Cloud Connect through Amazon Transit Gateway or IPsec.

See [DNS Filtering with emnify Cloud Connect and AWS Route53](https://www.emnify.com/developer-blog/dns-filtering) on the emnify Developer Blog for step-by-step instructions.
:::

## Add a custom DNS

1. [Log in to your emnify account](https://portal.emnify.com/sign).
2. Navigate to [**Device Policies**](https://portal.emnify.com/device-policies).
3. Find your desired service policy in the **Service Policies** section and click **Details**.

<img
  className="img"
  src={require('./assets/portal-device-policies-service-policies-details.png').default}
  alt=""
/>

4. Select the **More options** tab to find the **Custom DNS** section.
5. Click **Configure custom DNS**.

<img
  className="img"
  src={require('./assets/portal-device-policies-service-policies-details-more-options-configure-custom-dns.png').default}
  alt=""
/>

6. Enter the primary and secondary DNS, then click **Add custom DNS**.

<img
  className="img"
  src={require('./assets/portal-device-policies-service-policies-details-more-options-custom-dns-add.png').default}
  alt=""
/>

## Configure DNS settings with the emnify REST API

You can also add, list, or delete DNS configurations using the emnify Rest API.
For step-by-step instructions, see [Configure DNS Settings](https://cdn.emnify.net/api/doc/configure-dns-settings.html) in the emnify System Documentation.
